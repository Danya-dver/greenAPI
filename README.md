# GREEN-API Test Solution

This project is my solution for the DevOps test assignment.
I created a simple HTML page that works with GREEN-API. On the page you can enter idInstance and ApiTokenInstance, call API methods, and see the response in a separate read-only field. Everything works directly in the browser without any backend.
The page supports the following methods: getSettings, getStateInstance, sendMessage and sendFileByUrl. After entering the credentials, you can click the buttons and immediately see the API response in JSON format.
I tried to keep the implementation clean and simple. The layout follows the required mockup, and the right panel is fixed with internal scrolling so it удобно использовать even with large responses.
From a DevOps perspective, I focused on simplicity and deployability. The project does not require a server and can be easily hosted on GitHub Pages, Netlify or any static hosting. It can also be extended with CI/CD, Docker, or a small backend if needed.
I am interested in the DevOps position because I like building solutions that are easy to run, easy to deploy and easy to maintain. I pay attention not only to code, but also to how the application will be used and delivered.
This task shows how I approach work: keep things simple, make them work reliably, and think about production from the start.
