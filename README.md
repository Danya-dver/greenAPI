# GREEN-API Test Solution

This project is my solution for a DevOps test assignment.
I created a simple HTML page that interacts with the GREEN API. On the page, you can enter an `idInstance` and an `ApiTokenInstance`, call API methods, and view the response in a separate read-only field. 
Everything works directly in the browser without any backend. The page supports the following methods: getSettings, getStateInstance, sendMessage, and sendFileByUrl. 
After entering your credentials, you can click the buttons and immediately see the API response in JSON format. I tried to keep the implementation concise and simple. 
The layout matches the required design, and the right panel is fixed with internal scrolling, making it convenient to use even with large responses. 
From a DevOps perspective, I focused on simplicity and deployability. 
The project does not require a server and can be easily hosted on GitHub Pages, Netlify, or any static hosting service. 
If necessary, it can also be extended using CI/CD, Docker, or a small backend. 
I am interested in a DevOps role because I enjoy creating solutions that are easy to launch, easy to deploy, and easy to maintain. 
I pay attention not only to the code, but also to how the application will be used and delivered. 
This project demonstrates my approach to work: keeping things simple, ensuring they work reliably, and thinking about the production environment from the very beginning.
